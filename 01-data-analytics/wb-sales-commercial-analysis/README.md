# Коммерческий анализ продаж Wildberries

**Проблема:** по «сырому» еженедельному отчёту WB сложно быстро понять экономику: выручка, вычеты, логистика, где основные потери.

**Решение:** расчёт по строкам отчёта (CSV), сводные таблицы потерь и **одностраничное заключение** для решений продавца.

**Стек:** Python (`wb_analysis.py`, `sales_analysis_commercial.py`), pandas; исходники данных в репозитории обезличены по необходимости для публикации.

**Ценность:** типовой **коммерческий** кейс (не учебный датасет): связь метрик WB с управленческими выводами.

## Запуск

1. **Клонировать** репозиторий, перейти в `01-data-analytics/wb-sales-commercial-analysis/`.
2. **Окружение:** `python -m venv .venv` → активировать `.venv`, затем `pip install -r requirements.txt`.
3. **Запуск (dev):** `python wb_analysis.py` (остальные скрипты — по необходимости).

### Smoke-чеклист

- [ ] `pip install -r requirements.txt` без ошибок.
- [ ] `python wb_analysis.py` завершается без исключения (CSV на месте).
- [ ] В консоли/выводе есть сводные метрики по отчёту.

См. [`DATA_ANONYMIZATION.md`](./DATA_ANONYMIZATION.md) перед публикацией сырых CSV.

## Ограничения

- Данные отчёта принадлежат реальному кабинету — перед публикацией репозитория убедитесь в отсутствии чувствительных артикулов/имён.
- Выводы справочные; управленческие решения требуют учёта складских и прочих данных вне отчёта WB.
