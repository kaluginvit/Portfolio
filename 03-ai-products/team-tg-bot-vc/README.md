# Telegram-бот для командной работы

Простой Telegram-бот для управления общим списком задач команды. Бот позволяет добавлять задачи, просматривать их список и экспортировать данные в CSV-файл.

## 🚀 Возможности

- **/start** — приветствие и инструкции по использованию
- **/add** — добавление новой задачи в общий список
- **/list** — просмотр всех задач в виде текста
- **/list_csv** — экспорт всех задач в CSV-файл

## 🛠️ Установка и настройка

### Шаг 1: Установка Python

Убедитесь, что у вас установлен Python 3.8 или выше. Проверить можно командой:
```bash
python --version
```

### Шаг 2: Создание виртуального окружения

Виртуальное окружение изолирует зависимости проекта от других проектов на вашем компьютере.

#### Windows:
```bash
# Создание виртуального окружения
python -m venv venv

# Активация виртуального окружения
venv\Scripts\activate
```

#### Linux/Mac:
```bash
# Создание виртуального окружения
python3 -m venv venv

# Активация виртуального окружения
source venv/bin/activate
```

После активации в начале строки терминала появится `(venv)`.

### Шаг 3: Установка зависимостей

После активации виртуального окружения установите необходимые пакеты:
```bash
pip install -r requirements.txt
```

### Шаг 4: Получение токена бота

1. Откройте Telegram и найдите бота [@BotFather](https://t.me/BotFather)
2. Отправьте команду `/newbot`
3. Следуйте инструкциям: укажите имя и username для бота
4. Скопируйте полученный токен (выглядит как `1234567890:ABCdefGHIjklMNOpqrsTUVwxyz`)

### Шаг 5: Настройка переменных окружения

Бот настраивается через переменные окружения. Доступные переменные:

- **BOT_TOKEN** (обязательная) — токен бота от BotFather
- **DATABASE_PATH** (опциональная) — путь к файлу базы данных (по умолчанию: `tasks.db`)
- **LOG_LEVEL** (опциональная) — уровень логирования: `DEBUG`, `INFO`, `WARNING`, `ERROR` (по умолчанию: `INFO`)

#### Вариант 1: Использование файла .env (РЕКОМЕНДУЕТСЯ) ⭐

Самый удобный способ — создать файл `.env` в корне проекта:

1. Создайте файл `.env` в корневой папке проекта
2. Добавьте в него следующее содержимое:

```env
BOT_TOKEN=ваш_токен_от_BotFather_здесь
DATABASE_PATH=tasks.db
LOG_LEVEL=INFO
```

3. Замените `ваш_токен_от_BotFather_здесь` на реальный токен от BotFather

**Преимущества:**
- Не нужно устанавливать переменные в системе
- Удобно хранить настройки в одном месте
- Файл `.env` автоматически игнорируется Git (не попадёт в репозиторий)

⚠️ **Важно:** Файл `.env` добавлен в `.gitignore`, поэтому ваши секретные данные не попадут в Git.

#### Вариант 2: Системные переменные окружения

#### Windows (PowerShell):
```powershell
# Обязательная переменная - токен бота
$env:BOT_TOKEN="ваш_токен_здесь"

# Опциональные переменные (можно не указывать)
$env:DATABASE_PATH="tasks.db"           # Путь к БД
$env:LOG_LEVEL="INFO"                   # Уровень логирования

# Постоянная установка (для всех будущих сессий)
[System.Environment]::SetEnvironmentVariable('BOT_TOKEN', 'ваш_токен_здесь', 'User')
[System.Environment]::SetEnvironmentVariable('DATABASE_PATH', 'tasks.db', 'User')
[System.Environment]::SetEnvironmentVariable('LOG_LEVEL', 'INFO', 'User')
```
После постоянной установки перезапустите PowerShell.

#### Windows (CMD):
```cmd
# Обязательная переменная - токен бота
set BOT_TOKEN=ваш_токен_здесь

# Опциональные переменные (можно не указывать)
set DATABASE_PATH=tasks.db
set LOG_LEVEL=INFO

# Постоянная установка
setx BOT_TOKEN "ваш_токен_здесь"
setx DATABASE_PATH "tasks.db"
setx LOG_LEVEL "INFO"
```
После `setx` перезапустите командную строку.

#### Linux/Mac:
```bash
# Обязательная переменная - токен бота
export BOT_TOKEN="ваш_токен_здесь"

# Опциональные переменные (можно не указывать)
export DATABASE_PATH="tasks.db"
export LOG_LEVEL="INFO"

# Постоянная установка (добавьте в ~/.bashrc или ~/.zshrc)
echo 'export BOT_TOKEN="ваш_токен_здесь"' >> ~/.bashrc
echo 'export DATABASE_PATH="tasks.db"' >> ~/.bashrc
echo 'export LOG_LEVEL="INFO"' >> ~/.bashrc
source ~/.bashrc
```

**Важно:** `BOT_TOKEN` — обязательная переменная. Без неё бот не запустится. Остальные переменные имеют значения по умолчанию и являются опциональными.

### Шаг 6: Запуск бота

Убедитесь, что виртуальное окружение активировано (видите `(venv)` в терминале), и запустите:
```bash
python main.py
```

Если всё настроено правильно, вы увидите сообщения о запуске бота и его готовности к работе.

## 📖 Использование

### Команда /start
Приветствует пользователя и показывает доступные команды.

### Команда /add
Добавляет новую задачу в базу данных.

**Использование:**
1. Отправьте команду `/add`
2. Бот запросит ввод текста задачи
3. Введите текст задачи в ответном сообщении

**Пример:**
```
Вы: /add
Бот: ➕ Добавление задачи
     📝 Введите текст задачи:

Вы: Купить кофе для команды
Бот: ✅ Задача добавлена!
     📝 Текст: Купить кофе для команды
     👤 Автор: Ваше имя
```

Бот сохранит задачу с указанием вашего имени пользователя и времени создания.

### Команда /list
Выводит все задачи из базы данных в виде текста.

**Формат вывода:**
```
Задача #1
Текст: Купить кофе для команды
Автор: @username
Создана: 2024-01-15 10:30:00
```

### Команда /list_csv
Экспортирует все задачи в CSV-файл и отправляет его вам.

**Формат CSV:**
```csv
id,text,user,created_at
1,Купить кофе для команды,@username,2024-01-15 10:30:00
```

## 🗄️ База данных

Бот использует SQLite — легковесную базу данных, которая хранится в файле `tasks.db` в корне проекта.

**Структура таблицы `tasks`:**
- `id` — уникальный идентификатор задачи (автоинкремент)
- `text` — текст задачи
- `user` — имя пользователя Telegram, создавшего задачу
- `created_at` — дата и время создания задачи

Файл базы данных создаётся автоматически при первом запуске бота.

## ⚙️ Переменные окружения

Бот настраивается через переменные окружения. Вот полный список доступных настроек:

| Переменная | Обязательная | По умолчанию | Описание |
|------------|--------------|--------------|----------|
| `BOT_TOKEN` | ✅ Да | — | Токен бота от BotFather (обязательно для запуска) |
| `DATABASE_PATH` | ❌ Нет | `tasks.db` | Путь к файлу базы данных SQLite |
| `LOG_LEVEL` | ❌ Нет | `INFO` | Уровень логирования: `DEBUG`, `INFO`, `WARNING`, `ERROR` |

### Примеры использования переменных

**1. Использование файла .env (РЕКОМЕНДУЕТСЯ) ⭐**

Создайте файл `.env` в корне проекта:

```env
# Минимальная конфигурация
BOT_TOKEN=ваш_токен_от_BotFather

# Полная конфигурация
BOT_TOKEN=ваш_токен_от_BotFather
DATABASE_PATH=data/my_tasks.db
LOG_LEVEL=DEBUG
```

Бот автоматически загрузит переменные из файла `.env` при запуске благодаря библиотеке `python-dotenv`.

**2. Системные переменные окружения:**

**Минимальная конфигурация (только обязательное):**
```bash
# Linux/Mac
export BOT_TOKEN="ваш_токен"

# Windows PowerShell
$env:BOT_TOKEN="ваш_токен"
```

**Полная конфигурация:**
```bash
# Linux/Mac
export BOT_TOKEN="ваш_токен"
export DATABASE_PATH="data/my_tasks.db"
export LOG_LEVEL="DEBUG"

# Windows PowerShell
$env:BOT_TOKEN="ваш_токен"
$env:DATABASE_PATH="data/my_tasks.db"
$env:LOG_LEVEL="DEBUG"
```

### Логирование

Уровни логирования:
- `DEBUG` — самый подробный уровень, показывает все детали работы бота
- `INFO` — стандартный уровень, показывает основные события
- `WARNING` — только предупреждения
- `ERROR` — только ошибки

- Бот работает в режиме поллинга (постоянно опрашивает серверы Telegram)

## 🐛 Решение проблем

### Ошибка "BOT_TOKEN is not set"
Убедитесь, что переменная окружения `BOT_TOKEN` установлена и содержит правильный токен. После установки переменной перезапустите терминал.

### Ошибка при импорте модулей
Убедитесь, что вы находитесь в корневой директории проекта и виртуальное окружение активировано.

### Бот не отвечает на команды
Проверьте, что бот запущен и нет ошибок в терминале. Убедитесь, что токен правильный и бот не был удалён в BotFather.
