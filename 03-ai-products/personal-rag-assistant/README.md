# Диалоговый Telegram-бот с поиском по своим данным

## В чём суть проекта

Это учебный **персональный ассистент в Telegram**, который не только отвечает в чате, но и **помнит контекст** и умеет **работать с вашими документами**. Смысл — показать связку **RAG (retrieval-augmented generation)**: релевантные фрагменты подтягиваются из векторной базы, а языковая модель даёт ответ с опорой на них.

**Зачем это нужно:**

- **Долговременная память** — осмысленные реплики пользователя индексируются в Pinecone; бот может опираться на прошлые темы диалога.
- **«Свои данные»** — вы присылаете PDF, DOCX и другие поддерживаемые файлы; текст извлекается через **Docling**, режется на фрагменты, кладётся в ту же базу. После этого можно **спрашивать по содержимому** загруженных файлов.
- **Инструменты Haystack** — по запросу доступны факты из внешних API (кошки, Wikipedia, погода, изображения собак и т.д.), как в типовом агентном сценарии.

Учебный кейс по памяти LLM, RAG и поиску по своим данным. Подходит как демонстрация навыков: пайплайны Haystack, Pinecone, прокси к OpenAI-совместимому API.

## Возможности

| Область | Поведение |
|--------|-----------|
| Диалог | Ответы с учётом краткой истории чата и сохранённой памяти в Pinecone |
| Файлы | Конвертация документа в текст, индексация, краткое резюме одним предложением, вопросы по тексту |
| Команды | `/start`, `/help`, `/clear` — сброс памяти о пользователе в Pinecone |
| Статическая база | Отдельный скрипт [`ingest_knowledge_base.py`](ingest_knowledge_base.py) для индексации `.md`/`.txt` из папки [`knowledge_base/`](knowledge_base/) |

Стек: **Haystack** (агент и инструменты), **Docling** (разбор документов), **Pinecone** (векторное хранилище), **ProxyAPI** или другой OpenAI-совместимый endpoint для чата и эмбеддингов.

Подробнее по файлам и потокам данных — [`ARCHITECTURE.md`](ARCHITECTURE.md).

## Запуск

Из **корня проекта** предпочтительно:

```powershell
python main.py
```

Эквивалентно (тонкая обёртка): `python bot.py`. В Docker используется **`main.py`** (см. `Dockerfile`).

Опционально Docker (после заполнения `.env` на хосте):

```powershell
docker build -t personal-rag-assistant .
docker run --env-file .env personal-rag-assistant
```

### Чек-лист проверки

1. Текст в Telegram → осмысленный ответ (память и/или инструменты).
2. `/clear` → подтверждение очистки памяти.
3. PDF или DOCX → сообщения «Файл получен…», затем «Готово…», одно предложение резюме.
4. Вопрос по содержимому загруженного файла (в т.ч. на английском — как часто советуют в методичке).

## Виртуальное окружение (venv)

Создать venv:

```bash
python -m venv .venv
```

Активировать:

**Windows (PowerShell):**

```powershell
.venv\Scripts\Activate.ps1
```

**macOS/Linux:**

```bash
source .venv/bin/activate
```

## Установка зависимостей

```bash
pip install -r requirements.txt
```

Пакет `docling` нужен для разбора PDF/DOCX; при первом запуске может подгружаться модель.

## Переменные окружения

1. Скопируйте `.env.example` в `.env`.
2. Заполните значения:
   - `PINECONE_API_KEY`
   - `PINECONE_INDEX_NAME`
   - `PROXYAPI_BASE_URL` (например `https://openai.api.proxyapi.ru/v1` или `https://api.proxyapi.ru/openai/v1`)
   - `PROXYAPI_API_KEY`
   - `OPENAI_EMBEDDING_MODEL` — под индекс **3072** обычно `text-embedding-3-large`, под **1536** — `text-embedding-3-small`
   - `TELEGRAM_BOT_TOKEN`
   - опционально: `HAY_V2_MAX_FILE_MB`, `HAY_V2_USER_DOC_CHUNK_SIZE`

Размерность индекса Pinecone должна совпадать с моделью эмбеддингов: `text-embedding-3-large` → **3072**, `text-embedding-3-small` → **1536**.
